# portals-mk-hub

## 项目简介

portals-mk-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定的域名、网站或服务，仅作为通用页面存储与展示的集合点。

## 目录说明

```
portals-mk-hub/
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 页面所需的静态资源（CSS、JS、图片等）
├── docs/           # 相关文档或说明文件
└── README.md       # 本文件
```

- **pages/**：每个 HTML 文件对应一个独立页面，文件名通常反映页面内容或用途。
- **assets/**：存放页面依赖的公共资源，建议按类型或页面组织子目录。
- **docs/**：记录仓库使用方式、页面列表或变更日志等。

## 页面归档说明

- 所有页面均为纯前端 HTML，不依赖后端服务。
- 页面之间相互独立，无链接或数据共享。
- 归档内容以通用、静态为主，不包含动态交互或外部数据源。
- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并确保其引用资源路径正确。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/portals-mk-hub.git
   ```
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件即可查看页面效果。
3. 也可通过 GitHub Pages 或其他静态托管服务部署整个仓库。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或优化现有页面。
- 新增页面时，请保持文件命名清晰，避免重复或冲突。
- 资源文件请统一放入 `assets/` 目录，避免散落于仓库根目录。
- 提交内容应遵循通用、中立的风格，不包含特定商业、政治或敏感信息。
